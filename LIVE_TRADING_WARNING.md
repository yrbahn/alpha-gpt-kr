# ⚠️ 실전투자 경고 및 안내

## 🚨 중요한 경고

### 실제 돈으로 거래됩니다
- 이 시스템은 **실제 계좌**에서 **실제 주문**을 발생시킵니다
- **손실 가능성**이 있습니다
- 투자는 본인 책임입니다

### 백테스트 ≠ 실전
| 요소 | 백테스트 | 실전 |
|------|---------|------|
| 슬리피지 | 없음 | 있음 (가격 차이) |
| 거래비용 | 없음 | 있음 (수수료 0.015%) |
| 체결 | 즉시 | 지연 가능 |
| 유동성 | 무한 | 제한적 |
| 심리 | 없음 | 큼 |

**예상 수익률이 실전에서 그대로 나오지 않습니다!**

## 🛡️ 리스크 관리 필수

### 1. 손절매/익절 설정
```python
STOP_LOSS_PCT = -0.05    # -5% 손절
TAKE_PROFIT_PCT = 0.10   # +10% 익절
```

### 2. 포지션 사이징
```python
MAX_STOCKS = 5           # 최대 5종목 (작게 시작!)
MAX_INVESTMENT = 1000000 # 최대 100만원만
```

### 3. 정기 모니터링
- 매일 계좌 확인
- 손익 추적
- 알파 팩터 유효성 검증

## 💰 현재 계좌 상태

**예수금: 50,000원**

### ⚠️ 예수금 부족 경고
- 현재 5만원으로는 실질적인 분산 투자가 어렵습니다
- 최소 **100만원 이상** 권장
- 종목당 최소 10만원 이상 투자 권장

### 권장 사항
1. **추가 입금** 고려
2. 또는 **1-2종목만** 테스트
3. 또는 **모의투자**로 먼저 검증

## 🚀 실전 투자 시작

### 안전한 시작 방법

#### 1단계: 신호만 확인
```bash
python run_live_trading.py
# 선택: 1 (알파 신호만 확인)
```

#### 2단계: 소액 테스트
- 1-2종목만
- 최소 금액으로
- 손절매 타이트하게

#### 3단계: 점진적 확대
- 1주일 모니터링 후
- 성과 확인
- 점진적으로 종목/금액 증가

## 📞 문제 발생 시

### 주문 오류
1. **잔고 부족**: 예수금 확인
2. **가격 제한**: 상한가/하한가 확인
3. **거래 정지**: 종목 상태 확인

### 긴급 상황
1. **전체 매도**: `trader.sell_all()` (구현 필요)
2. **시스템 중지**: 스크립트 종료
3. **수동 개입**: HTS/MTS에서 직접 매도
